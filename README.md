Review Assigner
Сервис для автоматического распределения code review между разработчиками.

Быстрый старт

Предварительные требования

Перед началом убедитесь, что установлены:
Docker и Docker Compose
Make (обычно предустановлен в Linux/macOS)
Go 1.19+ (только для разработки)

Первый запуск за 3 шага

1. Клонируйте репозиторий
git clone <your-repo-url>
cd review-assigner

2. Запустите проект одной командой
make quick-run
Эта команда соберет и запустит все сервисы в фоне.

3. Проверьте статус
make status
Убедитесь, что все сервисы работают корректно.

Подробная инструкция

Базовые команды

Сборка и запуск:
make build - Сборка проекта
make run - Запуск в foreground режиме
make stop - Остановка всех сервисов

Разработка:
make dev - Локальная разработка 
make run-local - Локальный запуск (требуется установленный Go и PostgreSQL)

Тестирование:
make test - Запуск всех тестов
make test-cover - Тесты с покрытием
make quick-test - Быстрые тесты основных сервисов

Мониторинг:
make logs - Просмотр логов в реальном времени
make health - Проверка здоровья сервиса

Полная пересборка

Если возникли проблемы с зависимостями или кэшем:
make fresh - Полная очистка и пересборка

Настройка окружения

Проект использует настройки по умолчанию:
Порт приложения: 8080
База данных: PostgreSQL на порту 5432

Проверка работоспособности

После запуска откройте в браузере:
Health-check: http://localhost:8080/health
Swagger UI api: http://localhost:8080/docs/api/index.html (после make swagger)
тз openapi.yml: http://localhost:8080/docs/technicial-task/index.html (после make swagger)
